# 智能提醒助手 (Smart Reminder) - 产品需求文档 (PRD)

## 一、产品概述
**1.1 产品背景**
当前线上教育平台学生面临完成率不高的问题。约 40% 的学生当天未完成任务即离开 App。现有的定时（每日 18:00）且单一内容的 Push 提醒方式效果有限（打开率仅 8%，任务完成率 45%）。为了提升学生的学习连续性与任务完成率，需要一套智能化、个性化的提醒系统。

**1.2 核心目标**
- **业务指标提升**：将每日任务完成率从当前的 60% 提升至 75%。
- **互动指标提升**：推送打开率从 8% 提升至 15%。
- **用户体验优化**：从“冷冰冰的催促”转变为“有温度的学习搭子陪伴”，根据用户状态提供不同风格的文案触达。

## 二、用户画像与分层
通过分析学生的活跃状态和任务完成习惯，将用户分为以下三类，针对性下发不同语气的 AI 提醒文案：
- **自驱型（Active）**：每天主动完成任务，占 40%。
  - **触达策略**：采用**激励型 (Motivational)** 文案，主打成就感、荣誉感、连胜挑战（如：“哇！你是今天全站前 5% 的效率大师！继续保持连胜！”）。
- **被动型（Passive）**：需要提醒才会完成任务，占 35%。
  - **触达策略**：采用**同理心 (Empathetic)** 文案，主打安抚、理解、减轻压力（如：“知道你今天有点累，不过再坚持一下，完成 X 就可以好好放松啦！”）。
- **流失型（At-risk）**：频繁多次不打开、提醒也不来，占 25%。
  - **触达策略**：采用**幽默型 (Humorous)** 文案，主打趣味、警示、打破僵局（如：“滴滴！前方高能预警，你的作业君正在向你招手！”）。

## 三、核心功能模块详述

### 3.1 学生端：智学平台首页 (Dashboard)
学生登录后的主学习界面，清晰呈现当前的学习进度与数据。
- **用户信息与模拟切换**：
  - 顶部导航栏展示用户的个性化问候（例如：“你好，林晓 👋”）。
  - （测试环境特有）支持下拉菜单切换不同类型用户（被动型、自驱型、流失型）以演示不同情况。
- **今日学习任务列表**：
  - **交互式状态切换**：用户可点击任务前的圆圈图标快速标记任务完成/未完成。采用**乐观更新 (Optimistic UI)** 技术，确保操作即时响应。
  - **待办任务**：展示任务名称、所属课程（Badge）、预计耗时。支持悬停交互，圆圈图标会自动转换为勾选预览。
  - **已完成任务**：呈现置灰、划线状态，并带有绿色的成功勾选图标。
  - **数据同步**：任务状态的变化会实时同步至全站所有相关组件（进度环、完成率看板等）。
- **时间模拟调试器 (Simulator)**：
  - 支持**分钟级精度**的时间模拟（Hour:Minute），用于精准测试不同时间窗口的提醒逻辑。
  - **视觉反馈**：顶部显示当前模拟时间的 Badge，并伴有呼吸灯效果提醒当前处于非系统时间状态。
- **学习数据看板**：
  - **今日完成率**：实时计算（已完成任务 / 总任务 * 100%）。
  - **连胜天数**：展示连续完成任务的天数，强化用户黏性。

### 3.2 学生端：学习搭子小组件 (Study Buddy Widget)
浮动于页面右下角的全局提醒与陪伴组件。
- **迷你状态 (Minimized)**：
  - 悬浮 Bot 图标，若有未完成任务，右上角显示红色的待办数量红点角标。
- **展开状态 (Expanded)**：
  - **智能提示 (Nudge Message)**：根据当前用户的 Persona 获取专属语气（同理心/激励/幽默）的 AI 提醒文案，包含具体的任务名和耗时。当用户完成全部任务时，强制切换为激励祝贺文案。
  - **上下文关联按钮**：底部的“开始任务”按钮会自动关联当前优先级最高的待办任务名（如“开始任务：阅读小测”），点击效果与主列表完全同步。
  - **社交杠杆 (Social Leverage)**：
    *   **队友督促 (Peer Nudge)**：当用户完成全部任务后，面板会显示“队友还在努力中”，并提供“督促队友”按钮，可一键发送趣味提醒，利用社交压力促学。
    *   **队友上线提醒**：当组队队友（Team-mate）登录系统时，Widget 上方会弹出实时通知气泡（如“队友 张博 已上线！”），增强实时陪伴感。
  - **视觉反馈**：展示当前任务进度的环形进度条 (Progress Ring)，以及火苗图标的连胜徽章 (Streak Badge)。
  - **状态区分**：当所有任务完成时，面板的主题色会从蓝色（进行中）过渡为琥珀色/金色（完成、荣誉状态）。

### 3.3 智能提醒引擎 (AI Nudge Engine)
后台支撑智能化触达的核心逻辑：
- **AI 批量生成与缓存**：
  - 支持通过 AI 接口（当前为 Mock 模拟）批量生成不同风格的文案模板。
  - 模板中支持动态变量插值：`{user_name}`, `{streak}`, `{task_name}`, `{task_duration}`, `{buddy_name}`。
- **分层策略匹配算法**：
  - **自驱型 (Active)**：低频打扰，仅在关键时段（如 21:00）提供成就反馈。
  - **被动型 (Passive)**：窗口匹配，根据用户设定的 `preferredReminderTime` 提前 5-10 分钟进行温和提醒。
  - **流失型 (At-risk)**：高频触达，通过排行榜效应（对比队友进度）和幽默风格进行紧急召回。
- **数据兜底**：若用户无待办任务，强制下发鼓励/庆祝文案。若特定风格文案消耗完，具备默认 Fallback 机制。

### 3.4 管理后台：智能提醒引擎看板 (Admin Dashboard)
供运营和产品团队监控与配置策略的后台页面。
- **核心数据总览**：
  - **总文案生成量**：展示当前预生成的 AI 提醒文案库存。
  - **平均触达打开率**：监控当前大盘的打开情况及环比增长。
  - **任务完成率**：实时比对当前完成率与目标（75%）的差距。
- **A/B 测试漏斗分析 (A/B Test Stats)**：
  - 根据文案风格（同理心、激励、幽默）分别统计：**曝光量 (Views)**、**点击量 (Clicks)** 及 **点击转化率 (CTR)**。
  - 以直观的横向柱状图展示各风格文案的效果对比，为运营决策提供依据。
- **异步生成引擎控制 (Engine Control Panel)**：
  - **权重调节**：提供滑块组件 (Range Slider) 实时调整各个文案风格的下发流量比例（如设置同理心占比、幽默占比）。
  - **参数保存**：点击“保存权重配置”后生效至后台策略。
  - **手动触发生成**：一键调用大模型批量生成新一批提醒文案（用于补充语料库）。

## 四、数据指标体系与埋点
为了验证产品的效果，需要对以下核心事件进行埋点和指标回收：
- **Nudge 曝光事件 (`nudge_viewed`)**：记录下发的 `copy_id`, `user_id`, `tone`。
- **Nudge 点击事件 (`nudge_clicked`)**：记录从提示进入任务的动作，用于计算各个 Tone 的 CTR。
- **任务完成事件 (`task_completed`)**：核算当天的核心任务完成率。

## 五、技术选型与规范 (参考)
- **前端框架**：Next.js (App Router), React, Tailwind CSS v4, Framer Motion (用于组件微交互动画)。
- **UI 组件**：shadcn/ui, Lucide React (图标)。
- **状态与请求**：基于 SWR 的服务端数据 Hook；全局状态管理使用 Zustand (`widget-store`, `task-store`, `team-store`) 确保跨组件数据的实时一致性。
- **类型安全**：TypeScript 配合 Zod 进行数据校验 (`lib/types.ts`)。
